# toLink-Rag 项目信息

## 1. 项目名称

- `toLink-Rag`

## 2. 项目定位

- 面向文档解析、知识入库与检索增强的后端服务
- 提供文件解析、LLM 增强、向量检索及消息驱动的异步任务处理能力

## 3. 核心技术栈

- `FastAPI`
- `SQLAlchemy`
- `MySQL`
- `Redis`
- `MinIO`
- `Kafka / RabbitMQ`
- `Qdrant / Elasticsearch`

## 4. 当前核心业务域

- 文档解析任务接收与消费
- 原始文件下载、解析、增强、分块
- 解析状态持久化与对象存储回写
- LLM 文本、视觉与表格增强

## 5. 当前架构特征

- API 服务与 Kafka 消费者运行在同一 Python 进程生命周期中
- 解析任务通过 MQ 异步消费触发
- 文档解析链路混合同步库调用、异步编排和对象存储交互
- 数据库同时存在同步 ORM 访问方式与异步 Session 基础设施

## 6. 当前需求分析关注点

- 保持文档解析链路稳定性
- 降低长任务对 Kafka 消费稳定性的影响
- 规范线程池使用边界，避免不合理的跨线程数据库访问
