# 文档切片模块接口文档

## 1. 文档说明

本文档面向模块使用方，描述 `markdown_parser` 与 `splitter` 当前公开接口的输入、输出、调用方式与典型使用路径。

本文档不重复技术文档中的以下内容：

- 设计目标与演进背景
- 架构分层原因
- 算法设计细节
- 测试覆盖策略

对应模块路径：

- `src/core/markdown_parser`
- `src/core/splitter`

## 3. 核心数据结构

### 3.1 `MarkdownElement`

用途：表示扫描后的单个 Markdown 结构元素。

字段：

- `type: ElementType`
- `content: str`
- `start_line: int`
- `end_line: int`
- `metadata: dict[str, Any]`

常见 `metadata`：

- 标题元素：`heading_level`, `heading_text`
- 代码块元素：`language`
- 图片元素：`url`, `alt`

### 3.2 `ParseResult`

用途：表示 `MarkdownParser` 的统一输出。

字段：

- `elements: list[MarkdownElement]`
- `tables: list[TableRef]`
- `images: list[ImageRef]`
- `source_file: str | None`
- `remainder: str`

### 3.3 `Chunk`

用途：表示最终分片结果。

字段：

- `content: str`
- `start_line: int`
- `end_line: int`
- `metadata: dict[str, Any]`

当前常见 `metadata`：

- `source_file`
- `element_types`
- `chunk_index`
- `heading_trail`
- `split_strategy`
- `semantic_percentile`
- `semantic_threshold`
- `semantic_group_index`
- `semantic_subchunk_index`
- `context_prev_tokens_applied`
- `context_next_tokens_applied`
- `context_overlap_mode`

### 3.4 `EmbeddedChunk`

用途：表示已完成最终向量化的 chunk。

字段：

- `chunk: Chunk`
- `embedding: list[float]`
- `embedding_model: str | None`
- `cached: bool`

访问透传属性：

- `content`
- `metadata`

## 4. `markdown_parser` 接口

### 4.1 `MarkdownParser`

标准入口类。

#### 4.1.1 `parse`

```python
parse(text: str, source_file: str | None = None) -> ParseResult
```

输入：

- `text`：原始 Markdown 文本
- `source_file`：可选来源文件名

输出：

- `ParseResult`

说明：

- 这是最常用入口
- 会同时产出 `elements`、`images`、`tables`

#### 4.1.2 `parse_file`

```python
parse_file(filepath: str, encoding: str = "utf-8") -> ParseResult
```

输入：

- `filepath`：Markdown 文件路径
- `encoding`：文件编码

输出：

- `ParseResult`

#### 4.1.3 `parse_flat`

```python
parse_flat(text: str) -> list[MarkdownElement]
```

用途：

- 仅做结构扫描
- 不组织完整 `ParseResult`

#### 4.1.4 `parse_images`

```python
parse_images(text: str) -> list[ImageRef]
```

用途：

- 仅提取图片引用

### 4.2 `MarkdownScanner`

#### 4.2.1 `scan`

```python
scan(text: str) -> list[MarkdownElement]
```

用途：

- 将 Markdown 文本扫描为有序元素列表

返回元素类型覆盖：

- 标题
- 段落
- 列表
- 引用
- 代码块
- 表格
- 图片
- Front Matter
- Horizontal Rule
- Math Block

### 4.3 `ImageExtractor`

#### 4.3.1 `extract`

```python
extract(text: str) -> list[ImageRef]
```

用途：

- 提取 Markdown 和 HTML 图片引用

### 4.4 `ImageDescriber`

#### 4.4.1 初始化

```python
ImageDescriber(vision_client: VisionClient)
```

依赖：

- 调用方需提供 `VisionClient` 实现

#### 4.4.2 `process`

```python
process(parse_result: ParseResult) -> ParseResult
```

用途：

- 对 `ParseResult` 中的图片进行描述增强
- 结果会回填到元素内容中

注意：

- 当前为就地修改后返回

### 4.5 `TableDescriber`

#### 4.5.1 初始化

```python
TableDescriber(table_client: TableClient)
```

依赖：

- 调用方需提供 `TableClient` 实现

#### 4.5.2 `process`

```python
process(parse_result: ParseResult) -> ParseResult
```

用途：

- 对 `ParseResult` 中的表格进行总结增强
- 结果会回填到表格元素内容中

## 5. `splitter` 接口

### 5.1 `BaseChunker`

抽象接口，主要用于自定义分片策略。

#### 5.1.1 `chunk`

```python
chunk(elements: list[MarkdownElement], **kwargs) -> list[Chunk]
```

#### 5.1.2 `chunk_from_parse_result`

```python
chunk_from_parse_result(parse_result: ParseResult, **kwargs) -> list[Chunk]
```

#### 5.1.3 `achunk`

```python
achunk(elements: list[MarkdownElement], **kwargs) -> list[Chunk]
```

#### 5.1.4 `achunk_from_parse_result`

```python
achunk_from_parse_result(parse_result: ParseResult, **kwargs) -> list[Chunk]
```

### 5.2 `ASTAwareChunker`

用途：

- 执行纯结构规则分片

主要入口：

```python
chunk(elements: list[MarkdownElement], **kwargs) -> list[Chunk]
```

适用场景：

- 不需要语义切片
- 只想保留标题边界和结构化实体保护

### 5.3 `PercentileSemanticChunker`

用途：

- 对大文本块执行自适应语义切片

#### 5.3.1 初始化

```python
PercentileSemanticChunker(
    embedder,
    tokenizer,
    percentile: float = 95.0,
    min_chunk_tokens: int = 150,
    max_chunk_tokens: int = 512,
    overlap_tokens: int = 50,
    overlap_percentage: float | None = None,
    min_distance_gate: float = 0.25,
)
```

关键参数：

- `embedder`：段落级 embedding 客户端
- `tokenizer`：token 计数与截断工具
- `percentile`：动态阈值分位点
- `min_chunk_tokens`：最小块长度
- `max_chunk_tokens`：最大块长度
- `overlap_tokens` / `overlap_percentage`：重叠配置
- `min_distance_gate`：绝对最小距离门槛

#### 5.3.2 `split`

```python
async split(text_block: str) -> list[str]
```

用途：

- 输入一个超长文本块
- 输出语义切好的文本块列表

#### 5.3.3 `group_texts`

```python
async group_texts(texts: Sequence[str]) -> list[list[int]]
```

用途：

- 对多段文本做分组
- 返回索引分组，而不是直接返回拼接文本

适合：

- 上层 pipeline 二次编排

#### 5.3.4 `last_stats`

类型：

- `SemanticChunkingStats`

可用于读取：

- `atom_count`
- `distances`
- `threshold`
- `breakpoints`
- `fallback_used`

### 5.4 `StructuredSemanticChunker`

用途：

- 执行“结构分片 + 超长正文语义细分”的两阶段分片

#### 5.4.1 初始化

```python
StructuredSemanticChunker(
    semantic_chunker: PercentileSemanticChunker,
    heading_break_level: int = 3,

```

#### 5.4.2 `chunk`


## 6. 典型调用方式

### 6.1 只做 Markdown 解析

```python
from src.core.markdown_parser import MarkdownParser

parser = MarkdownParser()
result = parser.parse(markdown_text, source_file="demo.md")
```

### 6.2 结构分片

```python
from src.core.splitter import ASTAwareChunker, ChunkingEngine

engine = ChunkingEngine(chunker=ASTAwareChunker())
chunks = engine.process(markdown_text, source_file="demo.md")
```

### 6.3 两阶段分片

```python
from src.core.splitter import (
    ChunkingEngine,
    PercentileSemanticChunker,
    StructuredSemanticChunker,
)

semantic = PercentileSemanticChunker(embedder=embedder, tokenizer=tokenizer)
chunker = StructuredSemanticChunker(semantic_chunker=semantic)
engine = ChunkingEngine(chunker=chunker)

chunks = await engine.aprocess(markdown_text, source_file="demo.md")
```

### 6.4 解析结果增强后再分片

```python
from src.core.markdown_parser import MarkdownParser, ImageDescriber, TableDescriber
from src.core.splitter import ChunkingEngine, StructuredSemanticChunker

parser = MarkdownParser()
parse_result = parser.parse(markdown_text, source_file="demo.md")

parse_result = ImageDescriber(vision_client).process(parse_result)
parse_result = TableDescriber(table_client).process(parse_result)

chunks = await engine.aprocess_parse_result(parse_result)
```

### 6.5 最终向量化

```python
from src.core.splitter import ChunkEmbeddingPipeline

pipeline = ChunkEmbeddingPipeline(
    chunking_engine=engine,
    embedder=embedder,
    embedding_model="your-embedding-model",
)

embedded_chunks = await pipeline.aprocess(markdown_text, source_file="demo.md")
```

## 7. 使用约束

### 7.1 同步与异步入口不要混用

以下同步方法不能在活动事件循环中调用：

- `StructuredSemanticChunker.chunk()`
- `ChunkEmbeddingPipeline.embed_chunks()`
- `ChunkEmbeddingPipeline.process()`
- `ChunkEmbeddingPipeline.process_file()`
- `ChunkEmbeddingPipeline.process_parse_result()`

在异步环境中，应使用对应的 `async` 版本。

### 7.2 `splitter` 只依赖解析结果契约

调用方如果自定义 parser 替换实现，需要保证仍能产出兼容的：

- `ParseResult`
- `MarkdownElement`
- `ElementType`

### 7.3 LLM 增强客户端需由外部实现

以下抽象类默认不提供真实网络实现：

- `VisionClient`
- `TableClient`

调用方需自行实现并注入。

## 8. 返回值选择建议

如果你只需要：

- 结构化文档：用 `ParseResult`
- 检索分片：用 `Chunk[]`
- 入库前向量结果：用 `EmbeddedChunk[]`

推荐优先级：

1. 仅分析文档结构时，用 `MarkdownParser`
2. 需要切片但暂不向量化时，用 `ChunkingEngine`
3. 需要直接拿最终可入库结果时，用 `ChunkEmbeddingPipeline`
