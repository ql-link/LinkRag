# AGENTS Tree Sync

## 目标

保持根目录 [AGENTS.md](../../../../AGENTS.md) 中的“当前项目结构”树与仓库现状一致。

## 触发规则

在以下情况必须同步更新 `AGENTS.md`：

1. 新增目录或核心文件
2. 删除目录或核心文件
3. 移动或重命名目录/文件
4. 调整目录职责，导致树上的注释不再准确

如果只是修改文件内容、没有改变结构，一般不需要更新这棵树。

## 更新范围

`AGENTS.md` 中的树只维护：
- 整体目录骨架
- 核心文件
- 必要的简短注释

不要把它扩展成完整文件清单。保持“骨架 + 关键入口”的粒度。

## 同步步骤

1. 完成文件/目录变更后，重新检查当前仓库结构。
2. 对照 `AGENTS.md` 中的树，删除失效节点。
3. 补充新增节点，保持现有缩进和树形符号风格一致。
4. 如果目录职责发生变化，同时修正行尾注释。
5. 在最终回复中明确说明：已同步更新 `AGENTS.md` 目录树。

## 约束

- 不要为了“绝对完整”把所有测试文件、临时文件、IDE 文件都列出来。
- 继续省略 `.git`、`.venv`、`.pytest_cache`、`__pycache__` 等运行时或缓存目录。
- 优先保留用户当前已经确认过的展示风格，只做最小必要修改。
- 如果用户明确要求“不要更新 AGENTS.md”，则遵从用户要求。
